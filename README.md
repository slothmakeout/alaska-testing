# Alaska Testing

тесты для медвежачьего апи, написано на python + pytest + requests + allure

## Установка

### 1. Запуск тестируемого сервиса

```bash
brew install colima docker
colima start
docker pull azshoo/alaska:1.0
docker run -d -P --name test azshoo/alaska:1.0
docker ps
```

Команда `docker ps` покажет порт, на котором доступен сервис, например `0.0.0.0:32768->8091/tcp`. Надо запомнить этот порт

### 2. Настройка проекта

```bash
git clone https://github.com/slothmakeout/alaska-testing
cd alaska-testing
uv sync
```

Укажи актуальный порт в `config.py`:
```python
BASE_URL = "http://localhost:ТВОЙ_ПОРТ"
```

## Запуск тестов

```bash
.venv/bin/pytest -s tests/test_bears.py
```

Флаг `-s` включает вывод `print` в реальном времени.

## Запуск allure-отчёта

```bash
allure serve allure-results
```

Если allure не установлен:
```bash
npm install -g allure-commandline
```

## Тест-кейсы

### Создание медведя

1. Успешное создание и получение медведя по id, сверка всех полей
2. Создание только с полем имени (ожидаем 400)
3. Создание только с полем возраста (ожидаем 400)
4. Создание только с полем типа (ожидаем 400)
5. Создание с каждым из 4 типов: POLAR, BROWN, BLACK, GUMMY (параметризованный)
6. Создание с пустым JSON (ожидаем 400)
7. Создание с неизвестным типом PINK (ожидаем 400)
8. Создание с типом в другом регистре Black (ожидаем 400)
9. Создание с невалидным возрастом: -1, 0, 53, 100 (параметризованный, ожидаем 400)
10. Создание со строкой в поле возраста (ожидаем 400)
11. Создание с именем из одного символа
12. Создание с лишними полями в payload (surname)

### Получение медведя

1. Получение всех медведей (создаем двух, проверяем что оба id в списке)
2. Получение по несуществующему id (ожидаем 400)
3. Получение с буквами в id /bear/abc (ожидаем 400)

### Обновление медведя

1. Успешное обновление всех полей
2. Частичное обновление - только имя
3. Частичное обновление - только возраст
4. Частичное обновление - только тип
5. Обновление с пустым телом (ожидаем 400)
6. Обновление с невалидными данными (ожидаем 400)
7. Обновление по несуществующему id (ожидаем 404)

### Удаление медведя

1. Удаление всех медведей (создаем двух, проверяем что список пуст после удаления)
2. Удаление по id (проверяем что медведь недоступен после)
3. Удаление по несуществующему id (ожидаем 404)
4. Удаление всех когда база пуста
5. Удаление с буквами в id (ожидаем 404)
6. Повторное удаление того же id дважды (ожидаем 404 на второй)
